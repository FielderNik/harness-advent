# Веб-клиент Harness Advent

`harness-advent-web` — Vite + TypeScript интерфейс для безопасного создания и наблюдения за заданиями Harness Advent. Он обращается только к Ktor API на `/api/v1`; браузер не получает ключи моделей, не запускает команды и не имеет доступа к подключённым рабочим копиям.

## Запуск

Требование: Node.js 20+.

```bash
cp .env.example .env.local
npm install
npm run dev
```

По умолчанию клиент использует относительный URL API. В development Vite проксирует `/api` и `/health` на `http://127.0.0.1:8080`, поэтому Ktor не требует CORS-настройки. В production размести клиент за тем же origin/reverse proxy, что и API.

Если инфраструктура уже настроила CORS для отдельного trusted origin, можно указать базовый URL API в `.env.local`:

```env
VITE_API_BASE_URL=http://127.0.0.1:8080
```

Это не место для ключей, токенов или адресов поставщиков моделей. Сервер публикует только безопасный каталог профилей и моделей.

## Возможности текущего API

- health/status и безопасная ошибка недоступности сервера;
- список и создание задач, отмена и ожидание approval;
- SSE timeline с переподключением и polling fallback;
- добавление разрешённого проекта и запуск безопасного сканирования;
- серверный каталог профилей моделей без секретов;
- текстовые RAG-источники и отчёты из артефактов.

Diff, проверки, комментарии и внешняя публикация отображаются как явно недоступные, пока сервер не начнёт публиковать соответствующие артефакты и approvals. Клиент не имитирует эти действия локальным состоянием.

Перед добавлением путь проекта всё равно должен быть указан в `HARNESS_ALLOWED_PROJECTS` при запуске сервера. Форма не снимает это серверное ограничение.
