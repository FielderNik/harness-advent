# Рабочий процесс Codex

Codex используется как основной агент для развития Harness Advent. Эти правила делают его работу воспроизводимой и безопасной.

## Перед началом задачи

1. Прочитай `AGENTS.md`, относящуюся документацию и инструкции вложенного модуля.
2. Проверь текущую структуру, незакоммиченные изменения и границы задачи.
3. Для изменений архитектуры укажи, какие компоненты и контракты затрагиваются.
4. Если задача может вызвать внешний эффект — запуск команд над проектом, изменение файлов подключённого репозитория, комментарий или pull request — проверь, что на него есть разрешение пользователя.

## Во время работы

- Начинай с минимального проверяемого решения.
- Сохраняй границы между ядром, Ktor API, адаптерами, хранением и веб-клиентом.
- Для внешних систем используй интерфейс/порт и отдельный адаптер; не встраивай клиент конкретного провайдера в сценарий.
- Для RAG указывай источники, не выдавай предположение за факт и не отправляй закрытые данные в облако без разрешения.
- При запуске субагентов формулируй им цель, область файлов, критерий готовности и ограничения. Не поручай им публикацию внешних изменений без явного разрешения.
- Обновляй документацию вместе с контрактами, конфигурацией или архитектурными решениями.

## Завершение задачи

В результате сообщи:

- что изменено и зачем;
- какие проверки выполнены и их результат;
- что намеренно не реализовано или требует следующего этапа;
- есть ли ограничения, риски либо действия, ожидающие подтверждения пользователя.

Если проверки не запускались, укажи причину и команду для запуска.
